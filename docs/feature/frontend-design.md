# Frontend Dashboard

## Goal

React 대시보드로 프로젝트 관리, 스프레드시트 뷰어, 번역 실행, 검수 리포트를 제공한다.

## Tech Stack

- **Framework**: React 18+ (TypeScript) — because 컴포넌트 기반 UI에 적합
- **Build**: Vite — because 빠른 HMR, React 생태계 표준
- **Data Fetching**: TanStack Query — because 서버 상태 관리, 캐시, 폴링 내장
- **Styling**: Tailwind CSS — because 유틸리티 기반, 빠른 프로토타이핑

## Screens

1. **프로젝트 목록**: 카드 그리드 랜딩 페이지. 프로젝트 생성/선택.
2. **시트 뷰어**: 사이드바(시트 목록 + 시트 추가/삭제) + 테이블(번역 데이터). 소스 셀 포함 모든 셀 인라인 편집 가능. 행(key) 추가 + 체크박스 선택 일괄 삭제. 번역/업데이트/검수 실행 트리거. 테이블 헤더에서 시트 표시 언어 추가(프로젝트 풀에서 선택)/숨김. 언어 숨김은 CSV 데이터 유지, UI 표시만 제어.
3. **시트 설정**: 모달 다이얼로그. 시트 뷰어의 Settings 버튼으로 열림. 소스 언어는 프로젝트 언어 목록에서 dropdown 선택. 번역 스타일, 용어집 오버라이드, 글자수 제한, 자유 지시사항. 미설정 필드는 프로젝트 기본값 상속 표시.
4. **용어집 편집기**: 프로젝트 용어집 CRUD. 원어-번역어 쌍 관리.
5. **스타일가이드 편집기**: 프로젝트 번역 스타일/톤 관리.
6. **검수 리포트**: Reviewer 에이전트 결과 표시. 이슈별 필터링.
7. **언어 설정**: 사이드바 메뉴. 프로젝트 지원 언어 관리. Unity 호환 locale 프리셋(Tier 1: EFIGS+CJK 10개, Tier 2: pt-BR, ru 등 8개) + 커스텀 입력. 언어 삭제 시 영향받는 시트 수/번역 건수 경고 다이얼로그.

## Architectural Decisions

- **사이드바 + 콘텐츠 레이아웃** — because 프로젝트 진입 후 여러 섹션(시트, 용어집, 스타일가이드, 검수)을 빠르게 전환해야 함
- **사이드바에서 시트 관리** — because 시트 목록을 보면서 바로 추가("+" 버튼)/삭제(호버 시 아이콘)하는 것이 자연스러움. 시트 추가 시 이름 입력 다이얼로그, 삭제 시 키 건수 경고 다이얼로그. 생성 후 새 시트로 자동 이동.
- **체크박스 선택 + 일괄 행 삭제** — because 여러 키를 한 번에 삭제하는 경우가 빈번하므로, 각 행에 체크박스와 상단 "Delete Selected" 버튼으로 일괄 삭제. 확인 다이얼로그에서 삭제될 행 수 표시.
- **카드 그리드 프로젝트 목록** — because 다중 프로젝트 관리가 필요하고, 카드 형태가 프로젝트 상태를 한눈에 보여줌
- **비동기 작업 진행률 표시** — because 번역 job이 비동기이므로, 프론트엔드에서 상태 폴링 + 진행률 UI 필요
- **시트 설정을 모달 다이얼로그로 편집** — because 설정 항목이 5개 정도로 적고, 시트 데이터를 떠나지 않고 빠르게 열고 닫을 수 있음. 시트 뷰어 Settings 버튼으로 진입.
- **프로젝트 기본값 상속 표시** — because 시트별 오버라이드가 없으면 프로젝트 기본값을 placeholder로 표시하여, 사용자가 현재 적용 중인 값을 확인할 수 있도록 함.
- **Unity 호환 locale 프리셋 제공** — because 게임 번역에서 자주 사용하는 언어 조합이 정해져 있고, locale 코드를 수동 입력하면 오류 가능성이 높음. Tier 1(EFIGS+CJK 10개) + Tier 2(pt-BR, ru 등 8개) 프리셋으로 빠르게 추가.
- **프로젝트 레벨 언어 관리를 별도 화면으로 분리** — because 프로젝트 전체 언어 풀은 시트 뷰어보다 상위 개념이고, 언어 삭제 시 전체 시트 CSV 컬럼이 삭제되는 비가역적 작업이므로 별도 화면이 적합.
- **소스 언어를 dropdown으로 선택** — because 자유 텍스트 입력 대비 프로젝트 언어 목록에서 선택하면 오타/불일치 방지. 시트 설정 모달에서 프로젝트 언어 목록을 dropdown으로 제공.

## Constraints

- Must: 비동기 번역 job의 진행률을 사용자에게 표시
- Must: 소스 언어 셀 포함 모든 셀은 편집 가능
- Must not: 프론트엔드에서 데이터 소스에 직접 접근 (백엔드 API 경유)
- Must: 시트 삭제 시 삭제될 키 건수를 경고 다이얼로그에 표시
- Must not: 실행 중인 번역 job이 있는 시트의 삭제 허용
- Must: 행 삭제 확인 다이얼로그에서 삭제될 행 수를 표시
- Must not: 실행 중인 번역 job이 있을 때 행 삭제 허용

## Scope

**In scope (v0)**: 7개 화면 전체, 프로젝트 CRUD, 시트 추가/삭제, 행(key) 추가/일괄 삭제, 시트 설정 모달(소스 언어 dropdown/번역 스타일/오버라이드), 프로젝트 언어 관리(프리셋 + 커스텀) + 시트별 표시 언어, 번역 실행/모니터링
**Out of scope**: 실시간 협업, 다크 모드, 모바일 반응형

## v1 Additions

### Goal

폴링을 WebSocket으로 전환하여 진행률 실시간 수신, Job 이력 화면 추가.

### Screens (v1 추가)

8. **Job History (추가)**: 사이드바에 메뉴 추가. 프로젝트 전체 작업 이력 테이블. job 타입, 상태, 시트명, 시각 표시.

### Architectural Decisions (v1)

- **WebSocket으로 진행률 수신** — because v0 폴링(1.5초 refetchInterval)을 대체. TanStack Query 폴링 제거, WebSocket 메시지로 job 상태 업데이트.
- **Job 실행 중에만 WebSocket 연결** — because 항상 연결 유지 불필요. 트리거 시 열고 완료/실패 시 닫음.

### Constraints (v1 추가)

- Must: WebSocket 연결 실패 시에도 진행률 표시가 중단되지 않을 것

### Scope (v1)

**In scope**: WebSocket 진행률, Job History 화면
**Out of scope**: 다크 모드, 모바일 반응형, 실시간 협업
