# Game Translation Agent — Frontend Design

## Goal

번역 에이전트 시스템의 웹 대시보드. 다중 프로젝트 관리, 스프레드시트 조회/편집, 번역/검수 실행, 용어집/스타일가이드 관리를 제공한다.

## Tech Stack

- **React + TypeScript** — because 편집 가능한 테이블, 다중 페이지 라우팅 등 복잡한 UI 인터랙션 필요
- **FastAPI 백엔드** — because 에이전트 시스템이 Python이므로 동일 언어로 API 계층 구성
- **Vite** — because React 프로젝트의 사실상 표준 빌드 도구

## Architectural Decisions

- **2단계 네비게이션**: 프로젝트 카드 목록(랜딩) → 프로젝트 상세(사이드바+콘텐츠) — because 프로젝트 선택은 빈도가 낮고, 프로젝트 내 작업(시트/용어집/리포트 전환)은 빈도가 높음. 높은 빈도 작업에 사이드바를 할당.
- **시트 뷰어가 기본 화면**: 프로젝트 진입 시 첫 번째 시트의 데이터 테이블을 바로 표시 — because 가장 빈번한 작업이 번역 데이터 확인/편집.
- **인라인 셀 편집 → Google Sheets 반영**: 셀 클릭 시 즉시 편집, 포커스 아웃 시 자동 저장 — because 번역 팀이 웹에서 직접 수정할 수 있어야 함 (사용자 요구).
- **번역/검수는 비동기 실행 후 결과만 표시**: 실행 버튼 클릭 → 로딩 → 완료 후 결과 표시 — because 실시간 스트리밍 불필요 (사용자 결정). 복잡도를 줄임.
- **시트별 설정 화면 존재**: 각 시트마다 기준언어 오버라이드, 번역 스타일(단답형/문장형/스토리), 최대 글자수, 용어집 오버라이드, 자유 지시사항 설정 가능 — because 하나의 프로젝트 안에서도 시트마다 번역 맥락이 다름 (사용자 요구).
- **미니말/클린 디자인**: 흰 배경, 단순 레이아웃, 최소 장식 — because 개발자+번역팀 모두 사용하는 도구. 기능 우선.

## Constraints

- **Must**: 빈 셀을 시각적으로 구분 (하이라이트) — 번역이 필요한 셀을 즉시 식별
- **Must**: 검수 이슈가 있는 셀도 시각적 구분 — 품질 문제 즉시 식별
- **Must**: 프로젝트 카드에 번역 진행률 표시 — 전체 현황 파악용
- **Must not**: 번역 실행 중 테이블 편집 허용 — 동시 쓰기 충돌 방지
- **Must not**: 프론트엔드에서 Google Sheets API 직접 호출 — 모든 시트 접근은 FastAPI를 경유

## Screens (6개)

1. **프로젝트 목록** — 카드 그리드. 이름, 시트 수, 진행률, 미번역 건수, 검수 이슈 수.
2. **시트 뷰어** — 사이드바(시트 목록 + 메뉴) + 편집 가능 테이블. 상단에 기준언어 선택, 번역/검수 실행 버튼.
3. **시트 설정** — 기준언어 오버라이드, 번역 스타일 프리셋, 최대 글자수, 용어집 오버라이드, 추가 지시사항.
4. **용어집 편집** — 프로젝트 공통 용어집. 검색 + 인라인 편집 테이블.
5. **스타일 가이드 편집** — 언어별 톤/호칭/메모 + 공통 지시사항.
6. **검수 리포트** — 최근 결과 요약, 이슈 필터 테이블, 검수 이력.

## Scope

**In scope (v0)**:
- 6개 화면 전체
- 셀 편집 → Sheets 반영
- 번역/검수 실행 (비동기, 결과만 표시)
- 프로젝트별 용어집/스타일가이드 CRUD
- 시트별 컨텍스트 설정

**Out of scope**:
- 사용자 로그인/인증
- 다크 모드
- 모바일 반응형 (데스크톱 우선)

## v0 이후 검토 방향 (확정 아님 — v0 사용 경험 후 결정)

- 실시간 번역 진행 상태 (WebSocket 스트리밍)
- 번역 이력 diff 비교 뷰
- 다크 모드
- 검수 이슈 → 자동 수정 원클릭 적용
- 키보드 단축키 (스프레드시트 UX 개선)
- 다중 사용자 동시 편집 (협업)
